About Q and A

Q and A is an experimental knowledge discovery engine. It indexes only a few high traffic websites like Wikipedia, Stackoverflow and uses datastores like DBPedia to display interesting information. Right now the index is being updated - this takes a long time, and search results will be affected during this time. To return to the homepage, click here

Right now, the index is being updated, to the site is relatively slow. But once indexing is complete, the site can run on relatively modest hardward. The process acting as the server uses less than 10MB of ram when idle, and can handle a relatively large number of requests. The index is spread out over the disk like a "Trie", and can be used to look up over a relative large dataset without using much ram. The reason for this design is that server hosting (VPS) is much cheaper the lower the speced machine is, and the goal was to build a scalable system by having each node just be a machine with 1Core, 1GB of ram, but enough storage to accomodate the index.During indexing, we use a single core with 4GB of ram.

The current index has only wikipedia - because the current goal is to work on the query processor to answer interesting questions in human language. The prototype you see on this website just does a simple keyword lookup, and doesn't work if you make spelling mistakes, or enter incomplete phrases. The goal of the first part of this project, was just to build an index on modest hardware, so that it could be scaled up later, at low cost.
